# 虚拟试穿代码需求书

## 目标

在机器端商品详情页增加 V1 虚拟试穿能力：用户选择带 `tryOnSilhouetteUrl` 的商品变体后，可以打开全屏试穿页，通过视觉服务启动前台试衣会话，消费本机 HTTP MJPEG 预览流，并在预览画面上固定叠加该变体的试穿剪影图。

V1 只做固定叠加层，不做 AI 试穿、身体追踪、尺码推荐、截图、上传或帧分析。

## 已有能力

- 后端/管理端媒体字段已存在：
  - 产品展示图：`displayImageMediaAssetId`
  - 变体试穿图：`tryOnSilhouetteMediaAssetId`
  - 机器目录 URL：`coverImageUrl`、`tryOnSilhouetteUrl`
- 机器视觉配置已包含：`visionEnabled`、`visionWsUrl`、`visionRequestTimeoutMs`
- 中部正面摄像头由视觉服务或本地摄像头管理层独占物理设备。
- 前端不直接打开物理摄像头；前端只请求前台试衣会话并渲染视觉服务返回的预览流。
- 当前 `apps/machine` 未引入 VueUse，不要求新增 `@vueuse/core`。

## 需要使用的 API / 符号

- `useTryOnPreview`
- `openVisionTryOnSession`
- `machineStore.config.visionWsUrl`
- `vision.try_on.start`
- `vision.try_on.started`
- `vision.try_on.stop`
- `previewUrl`
- `tryOnSilhouetteUrl`
- 路由名建议：`virtual-try-on`
- 试穿页 query 建议：`variantId`

禁止使用：

- `navigator.mediaDevices.getUserMedia`
- `MediaStream`
- `HTMLVideoElement.srcObject`
- 默认摄像头 fallback

前端试衣不拥有摄像头硬件句柄。试衣预览失败时，只显示“虚拟试穿预览启动失败”，不要提示“摄像头被占用，请稍候”。

## 实现步骤

### 1. 商品详情页入口

在 `ProductDetailView.vue` 中：

- 根据当前选中变体读取 `tryOnSilhouetteUrl`。
- 只有存在 `tryOnSilhouetteUrl` 时显示“试穿”入口。
- 点击后跳转到试穿页，并带上当前 `catalogKey` 和 `variantId`。
- 返回后仍保留原商品详情和选中变体。

参考结构：

```ts
const selectedTryOnSilhouetteUrl = computed(() => {
  // 1. 找到当前选中 variant
  // 2. 返回 variant.tryOnSilhouetteUrl ?? null
});

async function openTryOn(): Promise<void> {
  // 1. 如果没有 selected variant 或 tryOnSilhouetteUrl，直接 return
  // 2. router.push({ name: "virtual-try-on", params: { catalogKey }, query: { variantId } })
}
```

### 2. 试衣预览 composable

使用 `useTryOnPreview.ts`，封装客户试穿页的前台试衣会话生命周期。

当前签名：

```ts
export function useTryOnPreview(): {
  previewUrl: Readonly<Ref<string | null>>;
  errorMessage: Readonly<Ref<string | null>>;
  isStarting: Readonly<Ref<boolean>>;
  startPreview: (input?: {
    catalogKey?: string;
    variantId?: string;
    silhouetteUrl?: string | null;
  }) => Promise<void>;
  stopPreview: (reason?: VisionTryOnStopReason) => Promise<void>;
};
```

函数体要求：

```ts
async function startPreview(input): Promise<void> {
  // 1. 先停止旧 try-on session
  // 2. 如果 silhouetteUrl 为空，设置“当前规格暂不支持虚拟试穿”
  // 3. 调用 openVisionTryOnSession(machineStore.config, { catalogKey, variantId })
  // 4. openVisionTryOnSession 通过 visionWsUrl 握手，并发送 vision.try_on.start
  // 5. 收到 vision.try_on.started 后保存 previewUrl
  // 6. 如果异步返回时 composable 已停止/页面已卸载，立即 stop 新 session
  // 7. 失败时设置本地错误，不直接打开摄像头
}

async function stopPreview(reason): Promise<void> {
  // 1. 递增请求序号，使迟到的 startPreview 结果失效
  // 2. 向视觉服务发送 vision.try_on.stop
  // 3. 清空 previewUrl
  // 4. 清空启动中状态
}
```

禁止：

- 不要调用 `getUserMedia`。
- 不要把 `previewUrl` 转成 canvas 或逐帧读取。
- 不要捕获、保存、上传、记录或传递任何视频帧。

### 3. 试穿页

在 `VirtualTryOnView.vue` 中：

- 从路由读取 `catalogKey` 和 `variantId`。
- 从目录 store 找到商品和当前变体。
- 校验该变体存在 `tryOnSilhouetteUrl`。
- 使用 `useTryOnPreview()`。
- `onMounted` 调用 `startPreview({ catalogKey, variantId, silhouetteUrl })`。
- `onBeforeUnmount` 调用 `stopPreview("route_leave")`。
- 全屏渲染 `<img :src="previewUrl">` 作为 MJPEG 预览，叠加固定居中的 `<img :src="tryOnSilhouetteUrl">`。
- 退出时停止前台试衣会话并返回商品详情页，保留 `variantId`。

参考结构：

```ts
const catalogKey = computed(() => String(route.params.catalogKey ?? ""));
const variantId = computed(() => String(route.query.variantId ?? ""));
const selectedVariant = computed(() => {
  // 1. 从 catalogStore 找商品
  // 2. 从 variantCandidates 找 variantId 对应变体
});
const silhouetteUrl = computed(
  () => selectedVariant.value?.tryOnSilhouetteUrl ?? null,
);

const { previewUrl, errorMessage, isStarting, startPreview, stopPreview } =
  useTryOnPreview();

onMounted(() => {
  // 1. 如果 silhouetteUrl 缺失，显示“当前规格暂不支持虚拟试穿”
  // 2. 否则 startPreview({ catalogKey, variantId, silhouetteUrl })
});

onBeforeUnmount(() => {
  stopPreview("route_leave");
});

async function exitTryOn(): Promise<void> {
  // 1. stopPreview("user_exit")
  // 2. router.push({ name: "product-detail", params: { catalogKey }, query: { variantId } })
}
```

### 4. 路由

## 验收标准

- 没有 `tryOnSilhouetteUrl` 的变体不显示试穿入口。
- 有 `tryOnSilhouetteUrl` 的变体显示试穿入口。
- 切换颜色/规格后，试穿入口按新变体实时更新。
- 点击试穿进入全屏试穿页，并保留所选 `variantId`。
- 试穿页通过 `vision.try_on.start` 启动前台试衣会话。
- 视觉服务返回 `vision.try_on.started.previewUrl` 后，试穿页使用该 MJPEG URL 渲染预览。
- 视觉服务不可用、摄像头不可用或协议不支持时显示本地错误，不调用 `getUserMedia`。
- 试穿页显示实时 MJPEG 预览，并固定居中叠加 `tryOnSilhouetteUrl`。
- 退出试穿或页面卸载时发送 `vision.try_on.stop` 并清空 `previewUrl`。
- 如果 `vision.try_on.started` 异步返回时页面已卸载，迟到的 session 也会立即 stop。
- 进入/退出试穿不创建订单、不预留库存、不影响结账状态、不改变库存。
- 前端不保存、不上传、不解码、不逐帧读取预览画面。
- 退出试穿返回原商品详情页，并保留原选中变体。
