# 软硬件通信文档V1.0

## 1. 摘要

- 本协议基于**一件商品购买**的出库操作，暂时不考虑多件商品同时出库。
- 上位机需根据指令控制下位机，但无需关注下位机的具体实现，达到强解耦效果。
- 上位机需**根据下位机心跳**标志，判断当前系统状态，出现故障需及时发出提示给后台管理进行维护。
- 下位机会对上位机指令在**200ms**内做出相应，超出该时间窗口可判定为超时。但上位机**不可连续发送**多条指令，否则下位机可能会被卡死。
- 完整一次出库流程为：**上位机发送出库指令 -> 下位机出库 -> 机械机构运送到出货口向上位机发送提示 -> 开门用户取货 -> 取货完成关门 -> 机械结构复位。**

## 2. 硬件接口

- 采用**串口（USB-TLL）**进行通信，波特率为**115200**，校验位**None**，数据位数**8**，停止位数**1**。
- 串口设备名（COMX）会动态变化，需要在设备管理器中手动设置固定串口，VID/PID需要等具体设备落实以后才能确定。
- 串口插上usb接口后是独占的，不需要软硬件复位。

## 3. 货道规则

- 基于目前规划，在包装出来后大概率会有改动。行数范围1-A，格数范围（1-6行）1 - 5、（7-A行）1 - 4。（注：此处行是从下往上递增，格数是从外至里依次递增）
- 行数与格数不存在映射关系，上位机只需要知道商品A放在第m行第n格（这个需要上位机有一个本地或者后台能设置存储商品存放信息的小软件）

| 行 (Row) | 范围描述 | 列 1 | 列 2 | 列 3 | 列 4 | 列 5 |
| -------- | -------- | ---- | ---- | ---- | ---- | ---- |
| 1 - 6    | 下层货道 | 有效 | 有效 | 有效 | 有效 | 有效 |
| 7 - 11   | 上层货道 | 有效 | 有效 | 有效 | 有效 | 无效 |

​

## 4. 通信协议

- 以**十六进制**格式（0x01）、uint8，进行通信，每个byte代表一个数据。

### 1. 上位机 -> 下位机

- **数据格式：**第一字节（帧头`0x55`），第二字节（行），第三字节（格），第四字节（CRC - 8校验两个数据字节，不包含帧头，具体示例见`第五节`）。

- **发送：**每次控制发送一次完整数据即可，若接收到错误信号，下位机会忽略上位机指令并反馈，上位机可重复发送。
- **握手：**下位机超出时间未响应，上位机可向下位机发送`[0x55, 0xA0]`尝试与下位机沟通，正常情况下位机会优先响应回复当前状态`[0x55, 0xAA/0xAB/0xAC/0xAF/0xE3/0xE6]`，若持续200ms未回复或连续回复错误3次则可判定下位机离线。
- **温湿度获取：**发送`[0x55, 0xB0]`获取温湿度，下位机会返回当前温湿度`[0x55, 0xB0, 温度值, 湿度百分比]`。温度单位**°C**，温度范围**-10°C-100°C**，温度值按`int8`类型解析；湿度单位**%RH**，湿度范围**0%-100%**，湿度值按`uint8`类型解析。`[0x55, 0xB0, 0x00, 0x00]`为保留哨兵，表示当前无有效温湿度样本，不表示真实的`0°C/0%RH`。
- **温湿度设置：**发送`[0x55, 0xB1, 温度]`可设置空调目标温度，温度即真实温度，协议可调范围为**-10°C-100°C**。若温度接受数据正常，下位机会返回上位机发送的相同数据，若异常返回`[0x55, 0xE1]`超出边界。业务应用和Web UI首版将可设置范围收紧为**18°C-30°C**。
- **空调开关：**发送`[0x55, 0xB2, 0x00/0xFF]`可控制空调开启与关闭（默认关闭），若下位机执行指令后，会返回相同数据。（注：`0x00`表示关闭，`0xFF`表示开启）

​

### 2. 下位机 ->上位机

- **接收：**当接收到上位机出库指令，若边界检查及CRC校验无误，发送`[0x55, 0x00]`，开始执行出库指令。若超过200ms未接收到可重复发送出库数据，若重复三次均未收到响应，则判定下位机故障。若出现其他情况见`6-4`。
- **故障（0xE ）：**若**超出边界**返回`[0x55, 0xE1]`，若**CRC校验错误**返回`[0x55, 0xE2]`，若中途出现**机械故障**返回`[0x55, 0xE3]`，若**系统正忙**返回`[0x55, 0xE4]`（见`6-2`），若**取货超时**返回`[0x55, 0xE5]`（见`6-5`），若**取货平台有杂物**返回`[0x55, 0xE6]`。

  > [!NOTE]
  >
  > E3、E6既可作为一次性故障响应，也可作为持续故障状态心跳。上位机无需区分帧来源，仅以当前系统进入故障状态处理。

- **事件（0xF ）：**当出库程序结束，货物运送到出货口，下位机**间隔50ms连续发送3次**`[0x55, 0xF0]`表示货物已运送到门口，即将出货（此时上位机应该有动画提示用户取货），系统复位到零点**间隔50ms连续发送3次**`[0x55, 0xF1]`，标志这一次出库结束。
- **心跳（0xA ）：**在系统正常运行时每0.5s下位机会向上位机发送一个心跳帧，若连续**2s**未接收到下位机心跳帧，则表示控制系统出现异常。其中，`[0x55, 0xAA]`表示下位机正常运行，`[0x55, 0xAB]`表示正在出库，`[0x55, 0xAC]`表示正在取货，`[0x55, 0xAF]`表示正在复位，复位之后空闲状态又会发`[0x55, 0xAA]`。
  - 当出现机械故障时，下位机会停止工作，并将心跳帧替换为`[0x55, 0xE3]`，此时系统出现硬件错误，应当立即停止继续工作。**E3 状态可通过按下硬件上的复位按钮退出。**
  - 当取货平台上有物品未取走，下位机会停止工作，并将心跳帧替换为`[0x55, 0xE6]`，此时系统无法正常工作，应当立即停止继续工作。**E6 状态可在清除异物后通过按下硬件上的复位按钮退出。**
- **温湿度数据：**当上位机发送获取温湿度数据时，下位机即在200ms内返回当前温湿度数据（数据格式见上）。若返回`[0x55, 0xB0, 0x00, 0x00]`，上位机应视为本次无有效样本且不更新最近有效温湿度；若连续3次均返回该保留哨兵，则判定温湿度传感器故障。

## 5. 校验规则

```TXT
示例：将第二行第五格出库
第一字节：0x55；第二字节：0x02；第三字节：0x05；
第四字节计算如下
1. crc = 0;
2. crc ^= data[i]
3. 循环8次：
   - 若最高位为1：
       crc = (crc << 1) ^ 0x07
   - 否则：
       crc <<= 1

示例：
数据：0x02 0x05
CRC结果：0x31

最终输出即为[0x55, 0x02, 0x05, 0x31]
```

对应C示例

```C
uint8_t crc8(const uint8_t *data, uint16_t len)
{
    uint8_t crc = 0x00;
    while(len--){
        crc ^= *data++;
        for(uint8_t i = 0; i < 8; i++){
            if(crc & 0x80) crc = (crc << 1) ^ 0x07;
            else crc <<= 1;
        }
    }
    return crc;
}
```

## 6. 补充

### 1. **心跳与业务帧交织**

- 有概率会在ACK前发送心跳，需要忽略心跳，同时下位机发送数据具有原子性，避免粘包情况发生。

- 上位机下发出货等关键控制指令后，需启动计时器。若在 **200ms** 内未收到有效的 ACK 响应，则触发重发机制；最多允许重试 **3次**。若连续 3 次重发均未收到 ACK，上位机即判定下位机通信离线，并停止后续指令下发，抛出离线告警。

### 2. **错误码恢复策略**

- **超出边界**、**CRC错误**允许立即重发控制指令，**机械故障**、**取货平台有杂物**需立即停止并向后台发出警告，**系统忙**至少间隔100ms再重发，不可连续发送指令。
- **忙碌**状态目前在初始化、出库、取货或复位的时候会触发，通常情况不会触发，上位机应根据心跳状态判断系统运行状态。该状态下接收到的出库指令均会被丢弃，若要出库需重新发送完整出库指令。

### 3. **多商品/多数量指令**

- 目前我的想法是扩展命令帧，`[0x55，第一件行，第一件格，第二件行，第二件格，... ，0x56，CRC-16]`，CRC还是校验数据段（不过实际硬件是否支持多件取货及最多能同时取多少件还需要验证）。
- 同样货道多件商品指令同上，按多商品处理。

### 4. **出货反馈信息意外丢失**

- 当发送出库指令，下位机返回的正常响应`[0x55, 0x00]`丢失，即上位机在200ms内未收到状态响应，应当发送`[0x55, 0xA0]`查询当前状态，若返回`[0x55, 0xAB/AC/AF]`则表示已正确收到指令。

### 5. **取货超时**

- 从柜门打开开始（返回状态`[0x55, 0xF0]`）后，下位机开始计时，并进入取货状态（心跳状态`[0x55, 0xAC]`）。
- 若用户在 **15s** 内未完成取货，下位机发送一次超时警告：`[0x55, 0xE5]`。此时上位机应提示用户尽快取走货物。
- 若用户在 **25s** 内仍未完成取货，下位机再次发送一次超时警告：`[0x55, 0xE5]`此时上位机可进一步提示用户设备即将自动关闭取货口。
- 若用户在 **30s** 内仍未完成取货，下位机将自动执行以下流程：关闭取货口、执行复位程序 、检测取货平台状态。
- 复位完成后，若检测到取货平台无遮挡、无残留货物或异物，则本次出货流程正常结束，发送：`[0x55, 0xF1]`并恢复空闲状态。
- 若复位完成后，检测到取货平台仍存在货物残留或异物遮挡，下位机发送故障信息：`[0x55, 0xE6]`，表示取货平台异常，需要人工介入处理。此时设备停止接收新的出货指令，并持续发送故障状态心跳；人工清除异物后，**按下硬件复位按钮**即可退出故障状态，恢复正常运行。

### 6.**并发操作**

- 系统任意时刻仅允许存在一个活动出库任务。当系统处于AB、AC、AF状态时，新的出库指令均返回E4。
- 温湿度查询(B0)、状态查询(A0)、空调控制(B1/B2)不受此限制。

### 7. **调试故障注入（仅用于测试）**

- **触发出库故障：** 在出库过程中，上位机向下位机发送 `[0x55, 0xFF, 0xFF, 0xFF]`，下位机将触发故障状态。此指令仅用于调试验收，不代表正常业务逻辑。
- **模拟机械故障（E3）：** 下位机固件在运行期间可随机主动发送 `[0x55, 0xE3]`，系统将直接进入机械故障死循环，心跳持续替换为 `[0x55, 0xE3]`，按下硬件复位按钮方可退出故障状态并恢复正常。

```TXT
上位机显示出库取货动画提示，开门取货；
4. 取货
下位机每隔0.5s发送心跳[0x55, 0xAC]；

  UI提醒即将关柜门；

回到状态1，下位机每隔0.5s发送心跳[0x55, 0xAA]
```

### 2. 异常状态

```TXT
1.CRC错误
上位机发送出库指令：[0x55, 0x02, 0x05, 0x32]；
									   ^
下位机接收到CRC错误，返回[0x55, 0xE2]；
下位机忽略指令，保持心跳[0x55, 0xAA]；
2.越界
上位机发送出库指令：[0x55, 0xFF, 0x05, 0xCC]；
                         ^^
下位机接收到超出范围，返回[0x55, 0xE1]；
下位机忽略指令，保持心跳[0x55, 0xAA]；
3.机械故障
下位机出库超时/复位超时，每隔0.5s发送机械故障警告[0x55，0xE3]；
此时需按下硬件复位按钮方可退出故障状态并重启下位机系统
4.忙状态
下位机正在执行出库或复位操作时，上位机发送出货指令，下位机会回复[0x55, 0xE4]（通常情况下不应该会触发）
5.心跳中断
    程序卡死、硬件故障等导致下位机死机，此时心跳中断，超过2s接收不到有效心跳帧上位机可尝试发送[0x55, 0xA0]，若200ms无回应再次发送，重复三次以上都无回应可判定下位机离线。

```

## 8. 指令与状态码汇总

### 1. 上位机 -> 下位机

| 指令                 | 数据格式                   | 说明                           |
| -------------------- | -------------------------- | ------------------------------ |
| 出库                 | `[0x55, Row, Col, CRC8]`   | 指定货道出库                   |
| 状态查询             | `[0x55, 0xA0]`             | 查询当前运行状态               |
| 获取温湿度           | `[0x55, 0xB0]`             | 获取当前温湿度                 |
| 设置温度             | `[0x55, 0xB1, Temp]`       | 设置空调目标温度               |
| 空调控制             | `[0x55, 0xB2, 0x00]`       | 关闭空调                       |
| 空调控制             | `[0x55, 0xB2, 0xFF]`       | 开启空调                       |
| 触发出库故障（调试） | `[0x55, 0xFF, 0xFF, 0xFF]` | 出库中触发故障状态，仅用于测试 |

### 2. 下位机响应码

**正常响应**

| 响应码       | 数据格式                       | 说明                   |
| ------------ | ------------------------------ | ---------------------- |
| ACK          | `[0x55, 0x00]`                 | 指令接收成功，开始执行 |
| 温湿度返回   | `[0x55, 0xB0, Temp, Humidity]` | 当前温湿度             |
| 温度设置成功 | `[0x55, 0xB1, Temp]`           | 温度设置成功           |
| 空调控制成功 | `[0x55, 0xB2, State]`          | 空调状态设置成功       |

**故障码（0xE）**

| 响应码 | 名称        | 说明                   |
| ------ | ----------- | ---------------------- |
| E1     | 超出边界    | 行列超出合法范围       |
| E2     | CRC校验错误 | CRC校验失败            |
| E3     | 机械故障    | 出库或复位失败         |
| E4     | 系统忙      | 当前无法执行新指令     |
| E5     | 取货超时    | 用户未及时取货         |
| E6     | 平台异物    | 平台存在货物残留或遮挡 |

**事件通知（0xF）**

| 响应码 | 名称       | 说明                     |
| ------ | ---------- | ------------------------ |
| F0     | 到达取货口 | 货物已送达，等待用户取货 |
| F1     | 出库完成   | 本次出库流程结束         |

**心跳状态码（0xA）**

| 状态码 | 状态   | 说明         |
| ------ | ------ | ------------ |
| AA     | 空闲   | 系统正常待机 |
| AB     | 出库中 | 正在执行出库 |
| AC     | 取货中 | 等待用户取货 |
| AF     | 复位中 | 正在执行复位 |

**故障状态心跳**

| 状态码 | 状态     | 说明                     |
| ------ | -------- | ------------------------ |
| E3     | 机械故障 | 系统停止运行，需人工处理 |
| E6     | 平台异常 | 平台有异物，需人工处理   |
