# 软硬件通信文档V1.1

## 1. 摘要

- 本协议基于**一件商品购买**的出库操作，暂时不考虑多件商品同时出库。
- 上位机需根据指令控制下位机，但无需关注下位机的具体实现，达到强解耦效果。
- 上位机需**根据下位机心跳**标志，判断当前系统状态，出现故障需及时发出提示给后台管理进行维护。
- ==上位机原则上**除指令外不发送多余信号**到通信总线上，下位机定时或根据指令返回当前的状态。==
- 下位机会对上位机指令在**200ms**内做出相应，超出该时间窗口可判定为超时。但上位机**不可连续发送**多条指令，否则下位机可能会被卡死。
- 完整一次出库流程为：**上位机发送出库指令 -> 下位机出库 -> 机械机构运送到出货口向上位机发送提示 -> 开门用户取货 -> 取货完成关门 -> 机械结构复位。**

## 2. 硬件接口

- 采用**串口（USB-RS485）**进行通信，波特率为**115200**，校验位**None**，数据位数**8**，停止位数**1**。
- ==上位机发送指令前，需判断当前通信总线上是否有其余设备在发送指令，如果在发送需等待至无信号之后才能发送指令。==
- 串口设备名（COMX）会动态变化，需要在设备管理器中手动设置固定串口，VID/PID需要等具体设备落实以后才能确定。
- 串口插上usb接口后是独占的，不需要软硬件复位。

## 3. 货道规则

- 基于目前规划，行数范围1-9，格数范围（1-6行）1 - 5、（7-8行）1 - 4、==（9行）1-3==。（注：此处行是从下往上递增，格数是从外至里依次递增）
- 行数与格数不存在映射关系，上位机只需要知道商品A放在第m行第n格（这个需要上位机有一个本地或者后台能设置存储商品存放信息的小软件）

| 行 (Row) | 范围描述 | 列 1 | 列 2 | 列 3 | 列 4 | 列 5 |
| -------- | -------- | ---- | ---- | ---- | ---- | ---- |
| 1 - 6    | 下层货道 | 有效 | 有效 | 有效 | 有效 | 有效 |
| 7 - 8    | 上层货道 | 有效 | 有效 | 有效 | 有效 | 无效 |
| 9        | 顶层货道 | 有效 | 有效 | 有效 | 无效 | 无效 |

## 4. 通信协议

- 以**十六进制**格式（0x01）、uint8（默认）进行通信，每个byte代表一个数据。

### 1. 上位机 -> 下位机

- **数据格式：**第一字节（帧头`0x55`），第二字节（行），第三字节（格），第四字节（CRC - 8校验两个数据字节，不包含帧头，具体示例见`第五节`）。

- **发送：**每次控制发送一次完整数据即可，若接收到错误信号，下位机会忽略上位机指令并反馈，上位机可重复发送。

- **握手：**下位机超出时间未响应，上位机可向下位机发送`[0x55, 0xA0]`尝试与下位机沟通，正常情况下位机会优先响应回复当前状态`[0x55, 0xAA/0xAB/0xAC/0xAF/0xE3/0xE6]`，若持续200ms未回复或连续回复错误3次则可判定下位机离线。

- **温湿度获取：**==发送`[0x55, 0xB0, 0x01]`获取出风口处（空调室）温湿度，发送`[0x55, 0xB0, 0x02]`获取外部温湿度，==下位机会返回当前温度`[0x55, 0xB0, 温度值, 湿度百分比]`，此处温度单位**°C**，温度范围**-10°C-100°C**，注意温度值应按`int8`类型看；此处湿度单位**%RH**，湿度范围**0%-100%**，湿度值对应`uint8`数据类型。

- **空调温度：**==发送`[0x55, 0xB1, 制冷/制热, 温度]`可设置空调温度（18 - 30）（默认25），制冷`0x00`/制热`0x01`（默认制冷），温度即真实温度，若温度接受数据正常，下位机会返回上位机发送的相同数据，若异常返回`[0x55, 0xE1]`超出边界。==

  - ==发送`[0x55, 0xB1]`可查询当前空调状态，下位机会返回`[0x55, 0xB1, 制冷/制热, 温度]`，可通过此方式实时更新当前状态==

  > [!NOTE]
  >
  > 1.  原理：空调具有制冷和制热的功能，当设定制冷温度为x、制热温度为y、环境温度为t、制冷温控范围为a~1~（默认3°C，且咱不向上位机开放设置接口）、制热温控范围为a~2~（与a~1~同理）。当t > x时，空调开启制冷；当t < x - a~1~时，停止制冷；当t从x - a~1~上升到x后，重新开始制冷，循环往复。当t < y时，开始制热；当t > y + a~2~时，停止制热；随后同上。因此上位机只需设置a~1~、a~2~值即可实现制冷与制热。
  > 2.  建议：机器开始制冷上位机端建议获取实时天气之后调整策略，比如气温高于23°C开启制冷，低于10°C开启制热，但制热功能最好不要开启。
  > 3.  注意：空调温度不可连续设置，期间间隔至少1s以上，否则可能会卡死控制器（通常设置1次即可）。当控制器死机时，下位机会启动硬复位（断电重启），期间需等待至少5s以上才能恢复对空调温度设置，中间都是系统正忙（E4）的错误码。

- **空调开关：**==发送`[0x55, 0xB2, 0x00/0xAA/0xFF]`可控制空调开启、软关闭与硬关闭（系统上电默认软关闭），若下位机执行指令后，会返回相同数据，若异常返回`[0x55, 0xE1]`超出边界；若距离上一次硬开关操作不足5s，下位机返回`[0x55, 0xE4]`。==

  - ==发送`[0x55, 0xB2]`可查询当前空调开关状态，下位机会返回`[0x55, 0xB2, 0x00/0xAA/0xFF]`。==

  > [!NOTE]
  >
  > 1. 说明：`0x00`表示开启，`[0xAA]`表示软关闭，`[0xFF]`表示硬关闭。开启即正常工作；软关闭是停止空调运行，但保持控制器上电；硬关闭是硬件上直接切断空调回路电源，实现绝对关闭。
  > 2. 建议：上位机指令控制开关建议是通过软关闭来实现空调关闭，除非长时间（比如一天都不需要）不使用空调，考虑到节能等因素直接切断电源，才使用硬关闭。
  > 3. 注意：硬关闭不能连续在开启和关闭的状态切换，期间至少间隔5s以上，否则触发系统正忙错误。

- **出风口**：==发送`[0x55, 0xB3, 风速]`即可控制出风口开关，当有人靠近时，发送该指令即可打开出风口和风扇，当人离开再次发送指令即可关闭出风口。发送`[0x55, 0xB3]`可查询当前状态（风速/出风口开合），下位机会返回`[0x55, 0xB3, 风速]`。==

  - ==风速这里提供五档：（出风口闭合）关闭`0x00`，（出风口开启）低速`0x01`，中速`0x02`，高速`0x03`，全速`[0x04]`。==
  - ==若下位机接收到正确的指令，会返回相同数据；若数据错误返回`[0x55, 0xE1]`超出边界；若系统正在开启或关闭途中，返回`[0x55, 0xE4]`。（上位机需设置保护阈值，当发送开启后5s内，不得再次发送其他出风口指令，否则视觉检测不准确在有人和无人之间跳跃时，容易造成硬件损坏）==

### 2. 下位机 ->上位机

- **接收：**当接收到上位机出库指令，若边界检查及CRC校验无误，发送`[0x55, 0x00]`，开始执行出库指令。若超过200ms未接收到可重复发送出库数据，若重复三次均未收到响应，则判定下位机故障。若出现其他情况见`6-4`。

- **故障（0xE ）：**若**超出边界**返回`[0x55, 0xE1]`，若**CRC校验错误**返回`[0x55, 0xE2]`，若中途出现**机械故障**返回`[0x55, 0xE3]`，若**系统正忙**返回`[0x55, 0xE4]`（见`6-2`），若**取货超时**返回`[0x55, 0xE5]`（见`6-5`），若**取货平台有杂物**返回`[0x55, 0xE6]`。

  > [!NOTE]
  >
  > E3、E6既可作为一次性故障响应，也可作为持续故障状态心跳。上位机无需区分帧来源，仅以当前系统进入故障状态处理。

- **事件（0xF ）：**当出库程序结束，货物运送到出货口，下位机**间隔50ms连续发送3次**`[0x55, 0xF0]`表示货物已运送到门口，即将出货（此时上位机应该有动画提示用户取货）；==~~系统复位到零点**间隔50ms连续发送3次**`[0x55, 0xF1]`，标志这一次出库结束。~~====客户取货完成，系统成功关闭取货门，下位机**间隔50ms连续发送3次**`[0x55, 0xF1]`；系统复位到零点**间隔50ms连续发送3次**`[0x55, 0xF2]`。==

- **心跳（0xA ）：**在系统正常运行时每0.5s下位机会向上位机发送一个心跳帧，若连续**2s**未接收到下位机心跳帧，则表示控制系统出现异常。其中，`[0x55, 0xAA]`表示下位机正常运行，`[0x55, 0xAB]`表示正在出库，`[0x55, 0xAC]`表示正在取货，`[0x55, 0xAF]`表示正在复位，复位之后空闲状态又会发`[0x55, 0xAA]`。

  - 当出现机械故障时，下位机会停止工作，并将心跳帧替换为`[0x55, 0xE3]`，此时系统出现硬件错误，应当立即停止继续工作。
  - 当取货平台上有物品未取走，下位机会停止工作，并将心跳帧替换为`[0x55, 0xE6]`，此时系统无法正常工作，应当立即停止继续工作。

- **温湿度数据：**当上位机发送获取温湿度数据时，下位机即在200ms内返回当前温湿度数据（数据格式见上），==若连续三次未收到有效温湿度数据或收到错误码E3，判定传感器异常。==

## 5. 校验规则

```TXT
示例：将第二行第五格出库
第一字节：0x55；第二字节：0x02；第三字节：0x05；
第四字节计算如下
1. crc = 0;
2. crc ^= data[i]
3. 循环8次：
   - 若最高位为1：
       crc = (crc << 1) ^ 0x07
   - 否则：
       crc <<= 1

示例：
数据：0x02 0x05
CRC结果：0x31

最终输出即为[0x55, 0x02, 0x05, 0x31]
```

对应C示例

```C
uint8_t crc8(const uint8_t *data, uint16_t len)
{
    uint8_t crc = 0x00;
    while(len--){
        crc ^= *data++;
        for(uint8_t i = 0; i < 8; i++){
            if(crc & 0x80) crc = (crc << 1) ^ 0x07;
            else crc <<= 1;
        }
    }
    return crc;
}
```

## 6. 补充

### 1. **心跳与业务帧交织**

- 有概率会在ACK前发送心跳，需要忽略心跳，同时下位机发送数据具有原子性，避免粘包情况发生。

- 上位机下发出货等关键控制指令后，需启动计时器。若在 **200ms** 内未收到有效的 ACK 响应，则触发重发机制；最多允许重试 **3次**。若连续 3 次重发均未收到 ACK，上位机即判定下位机通信离线，并停止后续指令下发，抛出离线告警。

### 2. **错误码恢复策略**

- **超出边界**、**CRC错误**允许立即重发控制指令，**机械故障**、**取货平台有杂物**需立即停止并向后台发出警告，**系统忙**至少间隔100ms再重发，不可连续发送指令。
- **忙碌**状态目前在初始化、出库、取货、复位、出风口及空调开关的时候会触发，通常情况不会触发，上位机应根据心跳状态判断系统运行状态。该状态下接收到的出库指令均会被丢弃，若要出库需重新发送完整出库指令。

### 3. **多商品/多数量指令**

- 目前我的想法是扩展命令帧，`[0x55，第一件行，第一件格，第二件行，第二件格，... ，0x56，CRC-16]`，CRC还是校验数据段（不过实际硬件是否支持多件取货及最多能同时取多少件还需要验证）。
- 同样货道多件商品指令同上，按多商品处理。

### 4. **出货反馈信息意外丢失**

- 当发送出库指令，下位机返回的正常响应`[0x55, 0x00]`丢失，即上位机在200ms内未收到状态响应，应当发送`[0x55, 0xA0]`查询当前状态，若返回`[0x55, 0xAB/AC/AF]`则表示已正确收到指令。

### 5. **取货超时**

- 从柜门打开开始（返回状态`[0x55, 0xF0]`）后，下位机开始计时，并进入取货状态（心跳状态`[0x55, 0xAC]`）。
- 若用户在 **15s** 内未完成取货，下位机发送一次超时警告：`[0x55, 0xE5]`。此时上位机应提示用户尽快取走货物。
- 若用户在 **25s** 内仍未完成取货，下位机再次发送一次超时警告：`[0x55, 0xE5]`此时上位机可进一步提示用户设备即将自动关闭取货口。
- 若用户在 **30s** 内仍未完成取货，下位机将自动执行以下流程：关闭取货口、执行复位程序 、检测取货平台状态。
- 复位完成后，若检测到取货平台无遮挡、无残留货物或异物，则本次出货流程正常结束，发送：`[0x55, 0xF2]`并恢复空闲状态。
- 若复位完成后，检测到取货平台仍存在货物残留或异物遮挡，下位机发送故障信息：`[0x55, 0xE6]`，表示取货平台异常，需要人工介入处理。此时设备停止接收新的出货指令，并持续发送故障状态心跳，直至人工排除故障并重新启动系统。

### 6.**并发操作**

- 系统任意时刻仅允许存在一个活动出库任务。当系统处于AB、AC、AF状态时，新的出库指令均返回E4。
- 温湿度查询(B0)、状态查询(A0)、空调控制(B1/B2)、出风口控制(B3)不受此限制。

### 3. 状态机流转

```
系统初始化
      │
      ▼
 AA（空闲）
```

## 8. 联调验收用例

### 1. 硬件系统初始化

```TXT
1. 下位机上电开始初始化。

2. 初始化期间，下位机不响应任何上位机指令（包括出货指令、状态查询、空调控制等），同时不会发送任何心跳帧。

3. 初始化完成后，下位机开始发送心跳帧：[0x55, 0xAA]
此时表示系统已进入正常工作状态，可接收上位机发送的各类指令。

4. 上位机首次检测到心跳后，可同步设备初始状态：

读取外部环境温湿度：
发送：[0x55, 0xB0, 0x02]
返回：[0x55, 0xB0, 0x20, 0x48]
                  32℃   72%

查询空调工作模式：
发送：[0x55, 0xB1]
返回：[0x55, 0xB1, 0x00, 0x19]
                  制冷   25℃

查询空调开关状态：
发送：[0x55, 0xB2]
返回：[0x55, 0xB2, 0xAA]
                 软关闭

查询出风口状态：
发送：[0x55, 0xB3]
返回：[0x55, 0xB3, 0x00]

5. 上位机获取当前温度状态，设置开启/保持关闭空调，若开启则设置空调温度
发送：[0x55, 0xB2, 0x00]
                  开启
返回: [0x55, 0xB2, 0x00]

发送：[0x55, 0xB1, 0x00, 0x14]
                  制冷   20°C
返回：[0x55, 0xB1, 0x00, 0x14]

6. 初始化时间应在10s以内。
若超过10s仍未收到任何有效心跳帧，则判定下位机初始化失败。
```

### 2. 完整出库流程

注：空调温度和出风量设置仅供参考

```TXT
1. 空闲状态
下位机初始化完成，默认空调为软关闭状态，默认制冷模式，设定温度25°C；
下位机每隔0.5s发送心跳：
[0x55, 0xAA]
---------------------------------------------------
2. 检测到顾客靠近
读取外部环境温湿度：
发送：[0x55, 0xB0, 0x02]
返回：[0x55, 0xB0, 0x20, 0x48]
                  32℃   72%

根据天气策略判断需要制冷。

开启出风口，中风：
发送：[0x55, 0xB3, 0x02]
返回：[0x55, 0xB3, 0x02]
----------------------------------------------------
3. 用户购买商品
假设购买第2行第5格。
发送：[0x55, 0x02, 0x05, 0x31]
下位机回复：[0x55, 0x00]
----------------------------------------------------
4. 出库过程
下位机进入出库状态，持续发送心跳：[0x55, 0xAB]。显示器播放取货等待动画。
货物运输至取货口，下位机间隔50ms连续发送3次：[0x55, 0xF0]
上位机播放出货动画并提示用户取货。
----------------------------------------------------
5. 用户取货
进入取货状态，下位机持续发送：[0x55, 0xAC]
若：
15s未取货，发送：[0x55, 0xE5]
UI提示："请尽快取走商品。"

25s未取货，再次发送：[0x55, 0xE5]
UI提示："柜门即将自动关闭。"

30s未取货：自动关闭柜门并执行复位。
关闭取货门后，连续发送3次：[0x55, 0xF1]。

若平台检测存在商品或异物：持续发送：[0x55, 0xE6]
----------------------------------------------------
6. 系统复位
下位机开始复位，持续发送：[0x55, 0xAF]
复位完成，连续发送3次：[0x55, 0xF2]
随后恢复：[0x55, 0xAA]
----------------------------------------------------
7. 用户离开
关闭出风口，发送：[0x55, 0xB3, 0x00]
返回：[0x55, 0xB3, 0x00]
系统恢复空闲，持续发送：[0x55, 0xAA]
```

### 3. 异常状态

```TXT
1.CRC错误
发送：[0x55,0x02,0x05,0x32]
下位机返回：[0x55,0xE2]
忽略此次出库。
继续发送：[0x55,0xAA]
----------------------------------------------------
2.越界
发送：[0x55,0xFF,0x05,0xCC]
下位机返回：[0x55,0xE1]
继续保持空闲心跳：[0x55,0xAA]
----------------------------------------------------
3.系统忙
系统正在AB、AC、AF状态。
再次发送出货指令：
下位机返回：[0x55,0xE4]
上位机等待状态恢复后重新发送完整出货指令。
----------------------------------------------------
4.机械故障
运行过程中发生机械故障。
下位机立即发送：[0x55,0xE3]
随后停止业务逻辑。
每0.5s持续发送：[0x55,0xE3]
需人工维修并重新启动系统。
----------------------------------------------------
5.取货平台异常
平台检测存在货物或异物。
持续发送：[0x55,0xE6]
停止接收新的出货任务。
待人工清理完成后重新启动系统。
----------------------------------------------------
6.心跳中断
连续2s未收到任何有效心跳。
发送：[0x55,0xA0]
200ms无响应：再次发送。
连续3次均无响应：判定下位机离线。
----------------------------------------------------
7.空调参数越界
发送：[0x55,0xB1,0x00,0x10]
温度低于18°C。
返回：[0x55,0xE1]
----------------------------------------------------
8.空调硬开关保护
连续5s内重复发送：[0x55,0xB2,0x00]
或[0x55,0xB2,0xFF]
下位机返回：[0x55,0xE4]
----------------------------------------------------
9.出风口保护
发送：[0x55,0xB3,0x03]
执行过程中再次发送：[0x55,0xB3,0x00]
下位机返回：[0x55,0xE4]
待运动完成后重新发送。
----------------------------------------------------
10.温湿度查询
发送：[0x55,0xB0,0x01]
200ms内返回：[0x55,0xB0,Temp,Hum]
若连续三次查询均失败，则判定温湿度传感器异常。
```
